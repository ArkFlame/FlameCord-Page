Frequently asked questions

How much RAM should I assign for FlameCord?

It depends on server specs, player count, and plugins, but 512MB is usually sufficient for most servers. A proxy does not require much RAM.

Can I use FlameCord with ViaVersion?

Yes, but you must place the ViaVersion jar in your Spigot server's plugins folder, not FlameCord's. This is essential for optimal resource usage. Download ViaVersion here.

Can I use FlameCord with Geyser?

Yes, FlameCord is compatible with Geyser. Be aware that Geyser itself is resource-intensive and may require a server with sufficient RAM. Download Geyser here.

I have an error in the console. What should I do?

Read the error message carefully to identify the cause (it often names a problematic plugin). If a plugin is at fault, try updating it or removing it. If you are unsure, seek assistance in our Discord Server.

Can I use gradients and hex colors in the MOTD?

Yes, you can use them in the MOTD using the IridiumColorAPI format. Examples can be found here in the usage section.

My server is crashing during an attack. What should I do?

First, check the crash logs for an "out of memory" error. If so, identify plugins causing high memory usage using a profiler like Spark. If the server isn't crashing but is unresponsive, you may need a hosting service with better DDoS mitigation or more network bandwidth.

My memory or CPU usage is high.

This is almost always caused by a plugin on your backend servers, not the proxy. Use the Spark plugin (/spark profiler and /spark heapdump) to identify the source of the high usage and address the problematic plugin.

I'm getting a "Too many accounts" error.

This means too many players are joining from the same IP address. If you're the owner, ensure you have enabled IP forwarding correctly on both your proxy and backend servers, especially if you use a reverse proxy like TCPShield. You can adjust the account limit in flamecord.yml.
